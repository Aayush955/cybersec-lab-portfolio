01 — About

Project Overview

A 3-month hands-on cybersecurity project simulating a real enterprise network environment from scratch — documented at every step.

🎯
OBJECTIVE

Build a fully virtualized enterprise-style security lab using KVM on a Linux host. Deploy pfSense, configure network segmentation, run IDS/IPS, perform penetration testing, and document everything as a professional portfolio.

🖥️
ENVIRONMENT

Host OS: Linux (KVM + virt-manager). Three isolated virtual networks: WAN (NAT), LAN (isolated), DMZ (isolated). All traffic routed through pfSense firewall acting as the sole gateway.

📅
TIMELINE

3 months total, working one dedicated day per week. Each session is documented with objectives, commands used, observations, issues encountered, and key security takeaways.

💼
PORTFOLIO GOAL

Demonstrate real-world security skills to prospective employers. Covers network architecture, firewall hardening, IDS/IPS, VPN, penetration testing methodology, and incident response.

03 — Roadmap

Lab Phases

10 phases covering the full security lifecycle — click any phase to expand the details and commands.

01
Environment Setup — KVM + virt-manager
✔ DONE

Installed and configured KVM hypervisor stack on Linux host. Verified CPU virtualization support, installed libvirt, added user to required groups.

KVMQEMU virt-managerlibvirt

Commands used this phase:

# Verify CPU supports virtualization
egrep -c '(vmx|svm)' /proc/cpuinfo
sudo kvm-ok

# Install KVM stack
sudo apt install -y qemu-kvm libvirt-daemon-system \
  libvirt-clients bridge-utils virt-manager

# Add user to groups
sudo usermod -aG libvirt,kvm $USER
sudo systemctl enable --now libvirtd

# Verify
virsh list --all
02
Virtual Network Design & Segmentation
✔ DONE

Created three isolated virtual networks using libvirt XML definitions. WAN uses NAT mode for internet access, LAN and DMZ are fully isolated so all traffic must pass through pfSense.

virshlibvirt XML Linux bridges
# Create all three networks
bash scripts/setup-kvm-networking.sh

# Verify networks are running
virsh net-list --all

# Confirm bridge interfaces on host
ip link show | grep virbr
brctl show

07 — GitHub

GitHub Setup

How this portfolio was set up and pushed to GitHub entirely from an Android phone using Termux.

1

Installed Termux on Android

Installed Termux from F-Droid (not Play Store — outdated version). Granted storage permissions to access phone Downloads folder.

termux-setup-storage
pkg update && pkg upgrade -y
2

Installed Git & GitHub CLI

All tools installed directly inside Termux on Android — no laptop needed.

pkg install -y git gh openssh nano
3

Configured Git Identity

Set name, email, and default branch. Used GitHub's no-reply email for privacy.

git config --global user.name "YOUR USERNAME"
git config --global user.email "your@email.com"
git config --global init.defaultBranch main
4

Authenticated GitHub CLI

Used browser-based OAuth flow. Opened github.com/login/device in Chrome, pasted the one-time code shown in Termux.

gh auth login
# Choose: GitHub.com → HTTPS → Login with browser
5

Pushed Portfolio Repo

Extracted the portfolio zip, initialized git, and pushed to GitHub as a private repo.

git init && git add .
git commit -m "Initial commit: portfolio structure"
gh repo create cybersec-lab-portfolio \
  --private --source=. --remote=origin --push
6

Created Profile README

Created the special username/username repo that displays on the GitHub profile page. Written in nano directly in Termux.

mkdir ~/Aayush955 && cd ~/Aayush955
git init && nano README.md
gh repo create Aayush955 --public \
  --source=. --remote=origin --push
★

Weekly Commit Routine

Every lab session — 4 commands to save and push all changes to GitHub.

cd ~/cybersec-lab-portfolio
git add .
git commit -m "[Phase N] What I did today"
git push origin main
